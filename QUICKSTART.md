# 快速开始指南

欢迎使用《神临山海》Novel as Code 创作环境！

## 🚀 立即开始

### 1. 在 Cursor 中打开项目

```bash
cd /Users/wyon/root/code-ai/神临山海
cursor .
```

### 2. 开始你的第一次AI协作

在 Cursor 的聊天框中，尝试以下命令：

#### 示例1：创建世界观设定
```
参考 @00_World_Bible/technology.md，帮我完善曲率引擎的详细设定，
包括能量消耗、安全限制、操作流程等。
```

#### 示例2：创建人物
```
根据 @00_World_Bible/society.md 中的社会结构，
帮我创建一个主角档案，要求：
- 出身于底层民众
- 通过努力成为技术人员
- 性格坚韧但内心有创伤
- 语言风格：简洁、直接、偶尔带点自嘲
```

#### 示例3：开始写作第一章
```
根据 @04_Outlines/overall_outline.md 和 @01_Characters/protagonist_main.md，
帮我写第一章的开头，要求：
- 展现主角的日常生活
- 暗示即将到来的危机
- 字数约2000字
- 风格冷峻、客观
```

#### 示例4：一致性检查
```
使用 Codebase 搜索，检查我在所有章节中提到的"曲率引擎"，
确保描述一致，没有矛盾。
```

### 3. 启动Web阅读器

#### 方法1：使用启动脚本（推荐）
```bash
./start-reader.sh
```

#### 方法2：手动启动
```bash
cd web-reader
python3 -m http.server 8000
```

然后在浏览器中打开：`http://localhost:8000`

## 💡 核心工作流程

### 日常写作流程

1. **准备阶段**
   - 打开相关设定文档（使用 `@文件名` 引用）
   - 回顾前文关键信息
   - 确认当前章节目标

2. **协作阶段**
   ```
   "根据 @01_Characters/protagonist_main.md 和 @00_World_Bible/technology.md，
   写一段主角操作曲率引擎的场景，重点描写技术细节和心理活动。"
   ```

3. **检查阶段**
   - AI会自动检查一致性
   - 你也可以手动要求检查：
   ```
   "检查刚才写的内容，确保人物性格和世界观设定一致。"
   ```

4. **修订阶段**
   - 根据AI建议修改
   - 更新相关设定文档（如有新设定）
   - 记录到 `04_Outlines/plot-threads.md`（如有新伏笔）

### 多文件协作示例

```
"主角在 @02_Drafts/Book1_灵机江湖/Chapter_001_入画.md 中受伤，
请同时更新：
1. @01_Characters/protagonist_main.md 的身体状态
2. @04_Outlines/plot-threads.md 记录这个事件的影响"
```

## 🎯 关键技巧

### 1. 使用 `@` 符号引用文档
```
@00_World_Bible/technology.md
@01_Characters/protagonist_main.md
```

### 2. 使用 Codebase 搜索
- 按 `Cmd/Ctrl + K` 打开搜索
- 或直接说："搜索全库，找到所有提到'黑曜石方尖碑'的地方"

### 3. 使用 Composer（多文件编辑）
- 按 `Cmd/Ctrl + I` 打开 Composer
- 可以同时编辑多个文件

### 4. 一致性检查
```
"检查 @02_Drafts/Book1_灵机江湖/ 中所有章节，
确保人物性格、世界观设定、时间线都一致。"
```

## 📝 重要提醒

1. **`.cursorrules` 是核心**：它定义了AI的行为规范，每次对话都会自动读取
2. **设定文档优先**：任何新设定都要先更新设定文档，再写入正文
3. **人物档案同步**：人物状态变化要及时更新档案
4. **伏笔管理**：所有伏笔都要记录在 `plot-threads.md`
5. **版本控制**：建议使用 Git 管理，可以像代码一样管理剧情分支

## 🔧 自定义配置

### 修改AI行为
编辑 `.cursorrules` 文件，可以：
- 调整写作风格
- 修改检查规则
- 添加特殊指令

### 自定义阅读器
编辑 `web-reader/styles.css` 可以：
- 修改颜色主题
- 调整字体大小
- 改变布局样式

## ❓ 常见问题

### Q: AI不遵守设定怎么办？
A: 确保使用了 `@文件名` 引用，或者明确说"参考XX文档"

### Q: 如何确保前后文一致？
A: 使用 Codebase 搜索功能，或明确要求AI检查一致性

### Q: Web阅读器无法加载章节？
A: 确保使用HTTP服务器（不能直接打开HTML文件），检查 `chapters.json` 中的路径

### Q: 如何管理多个剧情分支？
A: 使用Git分支功能，每个分支代表一个剧情方向
